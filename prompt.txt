🛠️ REVISIÓN DE CREDENCIALES DE CLIENTE EN KIDLINK-WEB (ERROR 401)Contexto del error:Al escanear el código QR en producción (https://kid-link.vercel.app/nino/[id]), la consola de Chrome arroja un error 42501 o 401 (Unauthorized) al consultar la API de Supabase en la ruta rest/v1/ninos_tags. Las políticas RLS de lectura pública ya están activas en PostgreSQL, por lo que el problema está en las cabeceras de la petición cliente.Instrucciones de inspección para el agente:Verificación de Prefijos: Revisa el archivo donde inicializas el cliente de Supabase (ejemplo: lib/supabaseClient.js o dentro de la ruta dinámica). Asegúrate de que estás utilizando estrictamente las variables con el prefijo de Next.js: process.env.NEXT_PUBLIC_SUPABASE_URL y process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY.Validación de Fallback: Añade un control de errores simple al inicializar el cliente que imprima un aviso en la consola del servidor si alguna de las dos variables llega como undefined, asegurando que no se intente realizar la consulta con parámetros vacíos.
